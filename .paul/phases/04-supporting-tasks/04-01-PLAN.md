## Goal
Create the two supporting task files: `tasks/status.md` (project pipeline view) and `tasks/add-type.md` (user-extensible type system).

## Purpose
Status gives visibility into what's in the ideation pipeline. Add-type makes SEED extensible — users can add new project types by creating data/ files without modifying any task code. Together they complete the command surface defined in seed.md.

## Output
- `tasks/status.md` (~60-80 lines)
- `tasks/add-type.md` (~80-100 lines)

  <action>
    **tasks/status.md** (~60-80 lines):

    Steps:
    1. `scan_projects` (priority="first") — Scan projects/ for directories containing PLANNING.md. For each, extract: name, Type metadata, creation date (from file), whether graduated (check for graduation note at bottom of PLANNING.md). Also check apps/ for graduated projects that still have projects/ remnants.

    2. `display_pipeline` — Present as a table:
       ```
       Project Pipeline
       ═══════════════════════════════════════
       Name         Type          Status
       ───────────────────────────────────────
       {name}       {type}        {Ready for /seed graduate}
       {name}       {type}        {Graduated → apps/{name}/}
       ═══════════════════════════════════════
       {N} projects | {M} ready to graduate
       ```
       If no projects found: "No projects in the pipeline. Run /seed to start one."

    **tasks/add-type.md** (~80-100 lines):

    Steps:
    1. `get_type_name` (priority="first") — Ask for the new type name (lowercase, hyphenated). Check data/{name}/ doesn't already exist. Wait for response.

    2. `define_config` — Ask about rigor level (tight/standard/deep/creative), demeanor description, and list required vs optional sections. Present the existing types as reference. Wait for response.

    3. `create_guide` — Based on the sections defined, generate data/{type}/guide.md with Explore + Suggest format for each section. Present draft for review. Wait for response.

    4. `create_config_and_loadout` — Create data/{type}/config.md with the rigor/demeanor/sections from step 2. Create data/{type}/skill-loadout.md — ask which ecosystem tools are relevant. Wait for response.

    5. `confirm_and_report` — Show what was created, confirm the type is now available for /seed ideate. No registration needed — composable data layer means the type exists as soon as the files do.

    **What to avoid:**
    - status.md should NOT modify any files — read-only scan
    - add-type should NOT require modifying seed.md or ideate.md — composable data layer means new types just work
    - Follow Explore+Suggest pattern established in Phase 3 guide files
  </action>
